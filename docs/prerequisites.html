Required — All Stacks
You need these to run any part of the project
required
Docker Desktop ≥ 26.0 required

Provides both the Docker engine and Docker Compose v2 (bundled). All three project stacks run entirely in containers — no local Python or Node.js install needed.

verify docker --version
verify docker compose version
install https://www.docker.com/products/docker-desktop/
⚠ Enable WSL 2 backend in Docker Desktop settings on Windows for best performance and hot-reload support.
Git ≥ 2.40 required

Standard version control. Used to clone the repository.

verify git --version
install https://git-scm.com/downloads
Make or PowerShell pick one

Makefile is used on Linux and macOS. dev.ps1 is the Windows equivalent — PowerShell 5.1+ ships with Windows 10/11, so no extra install needed on Windows.

Linux/macOS make --version
Windows $PSVersionTable.PSVersion
On macOS: brew install make  ·  On Ubuntu: sudo apt install make

Quick Start — After Installing Requirements
Clone → configure → run. All services start in Docker.
1
Clone the repository
git clone <repository-url> && cd plots
2
Create the environment file
Copy the template and fill in required values (see the Environment Variables section below).
Linux/macOS cp .env.example .env
Windows Copy-Item .env.example .env
3
Generate a SECRET_KEY
Paste the output into your .env file.
python -c "import secrets; print(secrets.token_hex(32))"
4
Start a stack
Choose the stack you want to run. All services start automatically.
monolith dev make dev-build
CQRS stack make cqrs-build
5
Run database migrations (monolith dev only)
Required on first start to create tables in PostgreSQL. CQRS services create their tables automatically via SQLAlchemy on startup.
make migrate

Kubernetes Deployment
Only needed if you're deploying to a Kubernetes cluster
optional
kubectl ≥ 1.28 k8s only

Kubernetes CLI for applying manifests in k8s/. Must match your cluster's server version within ±1 minor version.

verify kubectl version --client
docs https://kubernetes.io/docs/tasks/tools/
Helm ≥ 3.14 k8s only

Kubernetes package manager. Used to deploy infrastructure charts (Kafka, PostgreSQL, Zookeeper) into the plots-system namespace.

verify helm version
install https://helm.sh/docs/intro/install/
Kubernetes Cluster k8s only

Any CNCF-conformant cluster works. For local testing, use minikube or kind; for production, use GKE / EKS / AKS.

local — minikube minikube start --cpus 4 --memory 8192
local — kind kind create cluster --name plots
Kafka StatefulSets need at least 4 CPUs and 8 GB RAM on the cluster. Allocate accordingly in minikube.

External Services & Credentials
Enable these features by adding credentials to your .env
optional
Google OAuth 2.0 optional

Enables "Sign in with Google". Set AUTH_GOOGLE_ENABLED=true in .env to activate.

console console.cloud.google.com → APIs → Credentials → Create OAuth 2.0 Client
Add http://localhost/api/auth/google/callback to Authorised redirect URIs.
Set GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET in .env.
Never commit real credentials to git.
Azure AD (Microsoft SSO) optional

Enables enterprise SSO via Microsoft accounts. Set AUTH_AZURE_ENABLED=true.

portal portal.azure.com → Azure AD → App registrations → New registration
Add redirect URI: http://localhost/api/auth/azure/callback
Set AZURE_CLIENT_ID, AZURE_CLIENT_SECRET, AZURE_TENANT_ID.
SendGrid optional

Transactional email (password reset, account verification, notifications). Delivered directly by notification-service — no Celery required.

dashboard app.sendgrid.com → Settings → API Keys → Create API Key
Set SENDGRID_API_KEY and FROM_EMAIL. The sender address must be a verified sender identity in SendGrid.
Twilio optional

SMS notifications. Requires an active Twilio phone number with SMS capability.

console console.twilio.com → Account → API keys & tokens
Set TWILIO_ACCOUNT_SID, TWILIO_AUTH_TOKEN, and TWILIO_FROM_NUMBER.
Azure Blob Storage optional

File uploads (images, attachments). Without this, the uploads API returns errors for any file upload request.

portal portal.azure.com → Storage accounts → Create → Get connection string
Set AZURE_STORAGE_CONNECTION_STRING and AZURE_STORAGE_CONTAINER (default: uploads).
Stripe optional

Payments module. Currently a placeholder — the payments router is registered but no business logic is wired. Only needed when the payments feature is implemented.

dashboard dashboard.stripe.com → Developers → API keys

Minimum .env Configuration
Copy this into your .env file to get the dev stack running
config
.env — place in project root 
# ── Core ──────────────────────────────────────────────────────────
SECRET_KEY=<generate: python -c "import secrets; print(secrets.token_hex(32))">
DEBUG=true
ALLOWED_ORIGINS=http://localhost,http://localhost:3000,http://localhost:8000

# ── Database ──────────────────────────────────────────────────────
POSTGRES_USER=mvp_user
POSTGRES_PASSWORD=<choose a strong password>
POSTGRES_DB=mvp_db
DATABASE_URL=postgresql+asyncpg://mvp_user:${POSTGRES_PASSWORD}@postgres:5432/mvp_db

# ── Redis ─────────────────────────────────────────────────────────
REDIS_URL=redis://redis:6379/0

# ── Auth ──────────────────────────────────────────────────────────
AUTH_EMAIL_ENABLED=true
AUTH_GOOGLE_ENABLED=false
AUTH_AZURE_ENABLED=false
ACCESS_TOKEN_EXPIRE_MINUTES=30
REFRESH_TOKEN_EXPIRE_DAYS=7

# ── pgAdmin (dev stack only) ───────────────────────────────────────
PGADMIN_DEFAULT_EMAIL=admin@mvp.dev
PGADMIN_DEFAULT_PASSWORD=admin

# ── Optional: Google OAuth ─────────────────────────────────────────
# AUTH_GOOGLE_ENABLED=true
# GOOGLE_CLIENT_ID=<from Google Cloud Console>
# GOOGLE_CLIENT_SECRET=<from Google Cloud Console>
# GOOGLE_OAUTH_REDIRECT_URI=http://localhost/api/auth/google/callback

# ── Optional: Email (SendGrid) ─────────────────────────────────────
# SENDGRID_API_KEY=<your-key>
# FROM_EMAIL=noreply@yourapp.com

# ── Optional: File Uploads (Azure Blob) ───────────────────────────
# AZURE_STORAGE_CONNECTION_STRING=<from Azure Portal>
# AZURE_STORAGE_CONTAINER=uploads
